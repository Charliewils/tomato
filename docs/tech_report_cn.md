# 基于边缘AI的番茄温室智能监测系统——技术报告

> **项目类型**：学科竞赛作品 | **团队**：三人 | **周期**：2026年4月–7月
> **定位**：RV1126B端侧离线接地式番茄病虫害决策助手，NPU检测+LLM/CPU语音异构调度

---

## 一、项目概述

本系统以嵌入式AI开发板ELF-RV1126B为核心，集成计算机视觉、大语言模型Agent、语音交互、多源传感器和执行器控制，构建一套完全离线可运行的温室番茄智能管理系统。核心能力：

1. **实时病虫害检测**：YOLOv8n 13类目标检测（9种病害+3级果实成熟度），板端推理 ~25 FPS
2. **产量动态估算与预测**：基于检测框面积的MLP校准模型 + 时序预测器，输出当前估产和未来7天可采收量
3. **接地式语音Agent**：离线LLM注入实时数据，支持function calling——可读取传感器、控制风扇/水泵/补光灯
4. **环境多维感知**：三路I2C传感器（光照/土壤湿度/CO₂+温湿度），每5秒刷新
5. **智能执行器控制**：基于权威文献的病害感知气候管理——风扇（温湿度/VPD）、水泵（土壤湿度）、LED补光（DLI目标+光周期）
6. **数字孪生可视化**：React Three Fiber构建3D温室看板，含历史趋势图表，手机扫码即达

系统强调端侧离线、异构调度、Agent自动化——所有AI推理均在板端NPU/CPU完成，执行器可根据病害态势自动调整策略。

### 2.2 执行器

| 设备 | 引脚 | Linux GPIO | 控制方式 |
|------|------|:---:|------|
| 排风扇 | GPIO4_A1 | 129 | 继电器（光耦隔离），低电平有效 |
| 灌溉水泵 | GPIO3_B7 | 111 | 继电器（光耦隔离），低电平有效 |
| LED补光灯 | GPIO4_A0 | 128 | 软件PWM 500Hz → 光耦 → PWM板，低电平有效 |

所有执行器与GPIO间均有光耦隔离，GPIO切换加≥0.3s最小间隔保护，防止浪涌叠加。

### 2.3 I2C传感器总线

```
i2c-3 (21120000.i2c) —— ADS1115 ADC @0x48 —— 电容式土壤湿度探头
i2c-4 (21130000.i2c) —— VEML7700 光照 @0x10
                    —— SCD41 CO₂/温湿度 @0x62
```

## 三、传感器系统

### 3.1 环境光传感器 VEML7700

| 参数 | 规格 |
|------|------|
| 接口 | I2C，地址0x10，挂载i2c-4 |
| 量程 | 0–120k lux（5档自动量程） |
| 分辨率 | 0.0036 lux/count（最高灵敏档） |

### 3.2 土壤湿度传感器（电容式探头+ADS1115 ADC）

| 参数 | 规格 |
|------|------|
| ADC | ADS1115 16位ΔΣ，I2C地址0x48，挂载i2c-3 |
| 标定 | 干端(空气)2.772V→0%，湿端(泡水)1.540V→100% |

$$\text{soil\%} = \frac{2.772 - V_{soil}}{2.772 - 1.540} \times 100\%$$

### 3.3 CO₂/温湿度传感器 SCD41

| 参数 | 规格 |
|------|------|
| 测量原理 | 光声NDIR（CO₂）+ 片上CMOS（温湿度） |
| CO₂精度 | ±40 ppm + 5%读数 |
| CRC8 | 多项式0x31，初始值0xFF |

### 3.4 统一数据输出

```json
{
  "lux": 1045.1,  "soil": 62.5,  "co2": 715,
  "temp": 25.0,   "rh": 65.7,    "light": 8.0
}
```

## 六、离线大语言模型与Agent

### 6.1 LLM部署

| 项目 | 规格 |
|------|------|
| 基座模型 | Qwen2.5-0.5B-Instruct |
| 量化方案 | W4A16（RKLLM 1.2.3） |
| 推理速度 | 12.6 tok/s |
| 上下文 | 2048 tokens |

### 6.2 Agent工具体系

系统将LLM升级为Agent，支持6个工具（OpenAI function-calling兼容）：

| 工具 | 功能 |
|------|------|
| `get_sensors` | 读取温湿度/CO₂/光照/土壤 |
| `get_actuators` | 读取风扇/水泵/补光灯状态 |
| `get_plants` | 读取16株逐株记录 |
| `control_fan` | 开关风扇（排湿降温） |
| `control_pump` | 开关水泵（灌溉） |
| `control_light` | 控制补光灯开关及0-100%亮度 |

云端GLM-4-Flash使用原生function calling，本地0.5B使用`<tool>`标签解析降级。Agent最多3轮工具调用循环，工具结果注入后LLM生成最终回答。

### 6.3 接地式Prompt

动态注入实时传感器数据、逐株记录、室外天气，LLM引用具体数字作答。

## 八、智能执行器控制

基于10篇权威文献的病害感知气候管理策略（文献见附录）。

### 8.1 风扇控制

**P控制器 + 滞回**，目标值根据病害类型动态调整：

| 病害模式 | T范围 | RH范围 | VPD范围 | 策略 |
|----------|:-----:|:------:|:-------:|------|
| normal | 20-26°C | 60-80% | 0.7-1.2 | 最佳生长 |
| early_blight | 20-25°C | 55-75% | 0.8-1.5 | **降湿** |
| late_blight | 22-28°C | 50-70% | 0.8-1.5 | **升温+降湿** |
| leaf_mold | 20-25°C | 50-70% | 0.8-1.5 | **降湿** |
| spider_mites | 18-26°C | **65-85%** | 0.6-1.0 | **增湿！** |

**冲突解决**：红蜘蛛（需高湿>65%）与真菌（需低湿<75%）同时检出时取折中范围，priority机制加权决策。

### 8.2 水泵控制

土壤湿度迟滞控制：`soil < 40%` → 开泵灌溉，`soil > 70%` → 关泵。单次最长30s，冷却600s。模拟数据不做灌溉决策（`sim_soil=true`时跳过）。

### 8.3 LED补光灯控制

**基于DLI（Daily Light Integral）目标的PI控制器**：

| 参数 | 值 | 文献 |
|------|:--:|------|
| 目标DLI | 22 mol/m²/day | [R11] |
| 光周期 | 6:00-22:00 (16h) | [R11] |
| 触发阈值 | 环境 < 15 klux | [R12] |
| 关闭阈值 | 环境 > 25 klux | [R12] |
| 病害光补boost | 真菌类+15-20% | [R14][R15] |
| PWM频率 | 500 Hz | 光耦PC817额定1kHz |
| 调光速度 | 每周期 ≤10% | 防振荡 |
| 过渡方式 | ~400ms平滑渐变 | 防闪烁 |

**PPFD换算**：白光LED ≈ 1 klux → 14 µmol/m²/s。

## 十一、技术创新点

1. **全端侧离线Agent闭环**：检测→LLM Agent（6工具）→执行器控制，全部在RV1126B完成
2. **单NPU时分复用**：YOLOv8（8实例）与Qwen2.5-0.5B共享NPU，检测维持25fps
3. **文献驱动的病害感知控制**：基于10篇权威文献的温湿度/VPD/光照阈值表，根据实际检测到的病害自动切换控制策略
4. **DLI目标补光**：基于Daily Light Integral目标的PI控制器，病害时自动提升光照10-20%
5. **红蜘蛛/真菌冲突解决**：两类病害对湿度需求矛盾（高湿vs低湿），priority机制自动折中
6. **数字孪生历史趋势**：CSV历史数据→网页端果实/病害趋势图
7. **断电记忆**：选株记录plants.json落盘，重启自动恢复
8. **Agent工具链**：OpenAI function-calling兼容，云端原生/本地标签解析双模式
